# UI Rules

## Required Decisions

- Design token source
- Component naming convention
- Styling system
- Icon library
- Dark mode behavior
- Platform differences

## Rules

- Reuse existing design tokens and primitives before creating new ones.
- Components must expose meaningful loading, disabled, error, and accessibility states.
- Platform-specific UI differences must be intentional and documented.
- Avoid hard-coded colors and spacing when tokens exist.
- Keep reusable UI components product-agnostic.
